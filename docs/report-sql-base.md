**Три триггера которые делают умную работу за тебя:**

`trg_create_manager_profile` — как только регистрируется пользователь с ролью `manager`, профиль с балансами создаётся автоматически. Тебе не нужно думать об этом в коде.

`trg_commission_on_price` — ты ставишь цену в заказе, триггер сам считает `price × 0.3`, создаёт запись в `commission_transactions` со статусом `reserved` и обновляет баланс менеджера. Один PATCH-запрос — и всё произошло.

`trg_sync_balance` — при любом изменении статуса комиссии (`reserved → payable → paid`) балансы менеджера пересчитываются автоматически. Никакой ручной арифметики.

**RLS (Row Level Security)** — безопасность прямо на уровне БД. Менеджер физически не может получить чужие заявки даже если в API будет баг. Настраивается через `SET app.current_user_id = 'uuid'` в начале каждого запроса.

**`structured_brief JSONB`** с GIN-индексом — можно делать полнотекстовый поиск по брифам: `WHERE structured_brief @> '{"pain": "..."}' `. Удобно когда заявок станет много.

**`refresh_tokens`** — отдельная таблица для JWT refresh-токенов. При логауте просто удаляешь запись — и токен больше не работает.
