# Концепция агентства — v2.0

## Суть продукта

Веб-платформа с тремя ролями на старте: Владелец / Менеджер / Клиент. Единая экосистема где сходятся заявки, коммуникация, статусы и деньги. Менеджер приводит клиента и получает 30% комиссии — без найма, без зарплат.

---

## Роли и их кабинеты

### Владелец (ты)
- Дашборд всех заявок — от клиентов напрямую и от менеджеров
- Просмотр структурированного брифа по каждой заявке (AI уже обработал)
- Чат внутри заявки — отвечаешь на КП, задаёшь вопросы, даёшь рекомендации
- Смена статуса заявки
- Выставление цены → автоматически создаётся комиссия менеджеру
- Управление балансами менеджеров, фиксация выплат

### Менеджер
- Регистрация на сайте → роль "менеджер"
- Создание заявки: текст / голосовое (транскрипция Whisper) / форма-бриф
- LLM структурирует входящие данные в чёткий бриф автоматически
- Чат по заявке — видит ответы владельца, передаёт клиенту
- Баланс: зарезервировано → к выплате → выплачено
- Уведомления о статусах: в приложении + Telegram-бот

### Клиент (прямой)
- Отдельный флоу регистрации на сайте
- Описывает задачу текстом или голосом → LLM структурирует в бриф
- Заявка попадает напрямую к владельцу
- Получает КП, общается через чат заявки

## Жизненный цикл заявки

```
Новая → На рассмотрении → Отправлено КП → Договор подписан → В разработке → Завершён
                                                                             ↘ Отклонён
```

---

## AI-слой

- Голосовое сообщение → транскрипция через OpenAI Whisper
- Текст / транскрипт → GPT-4o структурирует в бриф:
  - Боль клиента
  - Текущие процессы (как сейчас решается задача)
  - Желаемый результат
  - Список функций
  - Интеграции
  - Бюджет / сроки (если упомянуты)
  - Уточняющие вопросы (генерирует GPT автоматически)
  - Краткое резюме для быстрого понимания
- Владелец видит готовый документ, не сырой текст

## Финансовая механика

| Событие | Что происходит |
|---|---|
| Владелец выставил цену | Менеджер видит "зарезервировано: X ₽ (30%)" |
| Клиент оплатил первый платёж | Комиссия → статус "к выплате" |
| Владелец нажал "выплатить" | Статус "выплачено", запись в истории |

Балансы менеджера обновляются автоматически через триггеры в БД.

## Стек

| Слой | Технология |
|---|---|
| Frontend + SSR | Next.js 14 (App Router) |
| Backend / API | Fastify (Node.js) |
| База данных | Supabase (PostgreSQL) |
| Auth | Supabase Auth (JWT, роли) |
| Хранилище файлов | Supabase Storage (аудио) |
| Realtime (чат) | Supabase Realtime |
| AI — транскрипция | OpenAI Whisper |
| AI — структуризация | OpenAI GPT-4o |
| Уведомления | Telegram Bot API + web push |
| Деплой | VPS (Hetzner) + Nginx + Docker |

## План разработки

### Этап 1 — Фундамент (2-3 дня)
- Монорепо: pnpm workspaces + turborepo
- Supabase проект: создать, накатить schema.sql
- Fastify сервер + config.ts (Zod валидация ENV)
- Next.js приложение, базовый роутинг
- Подключение Supabase клиента на фронте и бэке

### Этап 2 — Auth (1-2 дня)
- Supabase Auth: регистрация с выбором роли
- Разные формы регистрации под owner / manager / client
- Middleware на бэке: проверка Supabase JWT, извлечение роли
- Защищённые роуты на фронте, редиректы по роли
- Триггер автосоздания manager_profile

### Этап 3 — Заявки, ядро (4-5 дней)
- CRUD заказов (бэк): создание, список, детали, смена статуса
- Создание заявки менеджером — текстовый ввод
- Создание заявки клиентом напрямую
- Кабинет владельца: все заявки, фильтры по статусу
- Кабинет менеджера: свои заявки, статусы в реальном времени

### Этап 4 — AI-поток (2-3 дня)
- Загрузка аудио → Supabase Storage → Whisper транскрипция
- GPT-4o структуризация текста → structured_brief в БД
- Отображение брифа по полям в кабинете владельца
- Кнопка "переструктурировать" если бриф неточный

### Этап 5 — Чат и комиссии (2-3 дня)
- Realtime чат внутри заявки через Supabase Realtime
- Владелец выставляет цену → триггер создаёт комиссию 30%
- Кабинет менеджера: баланс (зарезервировано / к выплате / выплачено)
- Кабинет владельца: список комиссий, кнопка выплаты

### Этап 6 — Telegram-бот (1-2 дня)
- Привязка telegram_chat_id к аккаунту
- Уведомления менеджеру: смена статуса, ответ владельца
- Уведомления владельцу: новая заявка
- Webhook, не polling

### Этап 7 — Лендинг (2-3 дня)
- Публичная страница: боли, решения, портфолио
- CTA с разными флоу регистрации
- Адаптив под мобилу

### Этап 8 — Деплой (2 дня)
- Сборка Docker образов
- VPS: Nginx + SSL через Certbot
- CI/CD через GitHub Actions
- Мониторинг: UptimeRobot

**Итого: 3-4 недели** при активной разработке с LLM-агентом.

Критический путь: Этап 1 → 2 → 3 → 4. Этапы 6 и 7 можно параллельно после этапа 5.
